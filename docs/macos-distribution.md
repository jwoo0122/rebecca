# Rebecca macOS Distribution

## Development Requirements

- macOS 15.0+
- Xcode 26+ (or Xcode beta if using pre-release)
- Rust 1.85+
- xcodegen (`brew install xcodegen`)

## Building the App

```bash
# Build everything (Rust CLI + Swift host + resources)
./scripts/build-macos-app.sh

# Verify the built app
./scripts/verify-macos-app.sh dist/Rebecca.app
```

The build script:
1. Compiles the Rust CLI binary (`cargo build`)
2. Generates the Xcode project from `macos/project.yml` (if changed)
3. Builds the Swift host via `xcodebuild`
4. Copies resources (SKILL.md, icon, schema, CLI binary) into the bundle
5. Outputs `dist/Rebecca.app`

## Xcode Project

The Xcode project is generated by xcodegen from `macos/project.yml`.
Do not edit `Rebecca.xcodeproj` directly — edit `project.yml` and regenerate:

```bash
cd macos && xcodegen generate
```

### Icon (AppIcon.icon)

Icon Composer format (`.icon`) is supported on macOS 26+.
To add an icon:

1. Open the Xcode project: `open macos/Rebecca.xcodeproj`
2. Use Icon Composer to create/edit `macos/AppIcon.icon`
3. Reference it in `project.yml` if needed
4. Rebuild with `./scripts/build-macos-app.sh`

Until `AppIcon.icon` is added, the fallback `resources/Rebecca.icns` is used.

## Architecture

- **Host binary** (Swift): `Contents/MacOS/Rebecca` — runs as `LSUIElement` app, holds TCC permissions
- **CLI binary** (Rust): `Contents/Resources/bin/rebecca` — user-facing command-line tool
- **Socket**: `~/Library/Application Support/Rebecca/runtime/control.sock`

The CLI auto-launches the host app when the socket is unavailable. Before using
the socket, it checks the host bundle ID, executable path, and version. If an old
or development host owns the socket, the CLI terminates it and launches the
installed app explicitly. Set `REBECCA_APP_PATH` to use a development bundle.

## CI

### General CI (`.github/workflows/ci.yml`)
- Runs on every PR and push to main
- `cargo fmt --check`, `cargo clippy`, `cargo test`, `cargo build`
- Does not touch signing secrets

### Release CI (`.github/workflows/release.yml`)
- Runs from `main` for release-worthy Conventional Commits.
- `feat` creates a minor release; `fix` and `perf` create a patch release;
  `BREAKING CHANGE` creates a major release. Documentation, test, CI, and chore
  commits do not create a release.
- Builds, signs nested binaries and the app, notarizes, staples, and verifies the
  signed `Rebecca.app`.
- Creates the version tag and GitHub Release ZIP.
- Updates `jwoo0122/homebrew-tap/Casks/rebecca.rb` with the release SHA-256.

## Homebrew Cask

Install the signed and notarized app and its bundled CLI with:

```bash
brew tap jwoo0122/tap
brew install --cask jwoo0122/tap/rebecca
rebecca status
```

The release workflow uses the `HOMEBREW_TAP_TOKEN` secret to update the separate
`jwoo0122/homebrew-tap` repository.

## Permissions

Rebecca requires these TCC permissions:
- **Accessibility**: AXUIElement API (tree, find, press, set_value, window control)
- **Screen Recording**: ScreenCaptureKit (displays, windows, capture)

These are tracked by bundle ID (`dev.jwoo0122.rebecca`). Changing the bundle ID
or code signature invalidates existing permissions — users must re-grant.

Hardened runtime entitlements (`macos/Rebecca.entitlements`):
- `com.apple.security.automation.apple-events` — for NSWorkspace.openApplication

No other entitlements are needed. Input Monitoring is not required
(CGEvent uses `hidSystemState` which doesn't require it).

## Manual Verification Commands

```bash
# Build
./scripts/build-macos-app.sh

# Verify bundle
./scripts/verify-macos-app.sh dist/Rebecca.app

# Check signature (unsigned in dev)
codesign --verify --deep --strict --verbose=2 dist/Rebecca.app

# Check architecture
lipo -archs dist/Rebecca.app/Contents/MacOS/Rebecca

# Check plist
plutil -p dist/Rebecca.app/Contents/Info.plist
```
